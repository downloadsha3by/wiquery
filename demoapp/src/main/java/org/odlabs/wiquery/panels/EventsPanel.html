WiQuery events

WiQuery provides an extensive set of behaviors. Events are handled with behaviors.

Example 1: How events are handled

This first example shows how events are attached to a component. These events are attached in an unobtrusive fashion, by adding an event listener.

Click here to alert something

<span wicket:id="example1">Click here to alert something</span>

public MyWebPage() { super(id); Label label = new Label("example1", "Click here to alert something"); add(label); label.add(new WiQueryEventBehavior(new Event(MouseEvent.CLICK) { @Override public JsScope callback() { return JsScope.quickScope("alert('something :)')"); } })); }

Events are handled by three concepts:

WiQueryEventBehavior:
a class used to bind events with components
Event:
this class is part of WiQuery core and is used to create reusable events attached to multiple components. The constructor of the Event class takes a list of event labels. These event labels can be custom events.
JsScope:
this class abstracts JavaScript anonymous functions. A JsScope is like writing the following JavaScript statement:

function() { }
